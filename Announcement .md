## 3. Project Milestones and Core Requirements

To ensure a high-quality project, groups must follow these five stages.

### Stage 1: Background and Related Work

- **Introduction:** Briefly introduce the chosen application domain.
- **Algorithmic Summary:** Summarize the relevant algorithmic formulations.
- **Literature Review:** Discuss several representative papers.
- **Justification:** Explain why the chosen problem is algorithmically interesting.
- **Expectation:** Roughly 1-2 pages to demonstrate deep understanding.

### Stage 2: Representative Paper / Method Selection

- **Formulation:** Explain the formal problem formulation of the selected paper.
- **Algorithm:** Describe the core algorithm in detail.
- **Complexity:** Provide a rigorous analysis of the algorithm's complexity.
- **Objective:** Clearly define which experiments or results you intend to reproduce.

### Stage 3: Reproduction and Implementation

- **Implementation:** Correctly implement the chosen algorithm.
- **Results:** Reproduce at least part of the results reported in the original paper.
- **Analysis:** Explain any deviations or failures in the reproduction.
- **Performance:** Conduct an initial analysis of runtime and scalability.

### Stage 4: Experimental Evaluation

- **Runtime Analysis:** Measure and analyze execution times.
- **Scalability:** Evaluate how the algorithm performs as data size increases.
- **Comparison:** Compare your implementation with at least one baseline or variant.

### Stage 5: Optional Extension (Bonus-Level)

Outstanding projects may include extensions such as:

- Algorithm optimization or heuristic improvements.
- Approximation methods or better data preprocessing.
- Testing on alternative datasets or conducting additional experiments.

## 4. Submission Deliverables

### A. Project Proposal

A short document, roughly **1-2 pages**, that must include the following sections:

- **Topic Selection:** Chosen project topic, brief motivation, and application background.
- **Problem Formulation:** Clear definition of the computational problem, including input/output description and optimization objective.
- **Related Work:** Brief summary of at least two related papers and identification of the specific paper/method selected for reproduction.
- **Algorithm and Technical Plan:** Core algorithms involved, such as DP or graph algorithms, planned implementation approach, and expected datasets or evaluation setup.
- **Project Scope and Expected Goals:** What the group plans to reproduce and any possible optional extensions or improvements.
- **Team Information:** List of team members and expected division of contributions.

### B. Experimental Report

A comprehensive report based on the following project milestones:

- **Background and Related Work:** Introduction to the application domain, summary of algorithmic formulations, and discussion of representative papers.
- **Representative Paper / Method Analysis:** Explanation of the chosen paper's problem formulation, algorithm description, and complexity analysis.
- **Reproduction and Implementation:** Details of the algorithm implementation, reproduction of reported results, and explanation of any deviations or failures.
- **Experimental Evaluation:** In-depth analysis of runtime and scalability, including a comparison with at least one baseline or variant.
- **Optional Extension (Bonus-Level):** Documentation of any further optimizations, heuristic improvements, or experiments with alternative datasets.

### C. Source Code

The source code submission must include:

- Full implementation of the core algorithms.
- Documentation on how to run the reproduction experiments.
